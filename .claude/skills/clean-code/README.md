## clean-code

Clean code principles applied to banco-service: naming conventions (nouns for classes, verbs for methods, is/has/can for booleans, SCREAMING_SNAKE_CASE constants), method length and single responsibility, Optional handling without .get(), abstract exception hierarchies (never non-abstract base, never sealed), and eliminating magic numbers/strings with named constants.

Use this skill when naming anything, structuring methods, handling Optional, or building an exception hierarchy.
